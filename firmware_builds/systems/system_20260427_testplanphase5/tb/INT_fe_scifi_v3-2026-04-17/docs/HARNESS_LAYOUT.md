# Harness Layout

This INT harness is a source-level bridge between the current Qsys integration
and the future UVM environment.

Current structure:
- `tb_src/`: stable repo-local VHDL/SystemVerilog stubs and datapath-native
  smoke benches
- `generated/`: temporary v3 bench derivatives and quiet wrappers generated by
  the helper scripts
- `scripts/`: runnable entry points plus transform/build helpers
- `cases/`: domain-local stimulus inventory and future expansion buckets
- `REPORT/`: runtime evidence, split by domain
- `work/`: simulator libraries and build scratch

Why this split exists:
- source benches remain reviewable and diffable
- generated bench transforms stay isolated from simulator work products
- domain case planning is visible before the future UVM plan lands
- RC, SC, and DP evidence does not mix into one flat report bucket

Stable seams already proven useful:
- RC: upload-side `runctl_mgmt_host_*` seam and upper/lower frame-assembly seams
- SC: SC-hub external AVMM ingress and control/datapath address-map apertures
- DP: embedded emulator CSR seam plus exported datapath histogram AVMM window

Expected next step:
- keep the current INT harness as the fast source-integration regression
- derive UVM sequence buckets from `cases/*.md`
- preserve the same observable seams in the UVM env so current smoke evidence
  remains comparable to the future constrained-random harness
