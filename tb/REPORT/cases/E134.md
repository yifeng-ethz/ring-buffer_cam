## Intent

- **Scenario:** concurrent same-sector push_write + pop_erase forbidden: arrange a deassembly entry whose target sector matches the active `pop_issue_addr` sector and offer it on the `pop_erase_req=1` cycle
- **Primary checks:** `push_write_sector_locked=1` because the pop-issue extra lock added that sector; `decision_reg` reads `2` (pop-only), never `5`; on the next cycle the same entry is still blocked while `pop_issue_addr_pending` keeps the sector locked through emit
